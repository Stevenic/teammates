## Design

**Solve the user problem, not the screenshot**
A design is correct when the workflow is understandable, recoverable, and efficient, not when a single frame looks polished.

**Hierarchy does the heavy lifting**
Typography, spacing, and contrast should explain what matters before any decorative flourish is added.

**Constraints create coherence**
Use a small, intentional set of layout, motion, and color rules so the interface feels authored rather than assembled.

**Prototype edge states**
Empty, loading, error, and cramped layouts reveal whether the design system is real or just a happy-path mockup.
