## Documentation

**Write for the next action**
Good docs help the reader do something concrete: choose, install, fix, or verify.

**Examples carry the load**
Short, correct examples usually teach faster than long explanation blocks.

**Keep docs aligned with reality**
Whenever behavior, commands, or defaults change, update the docs in the same pass or call out the gap explicitly.

**Structure for scanning**
Users should be able to find prerequisites, steps, outputs, and caveats without reading every paragraph.
