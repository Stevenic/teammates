### Key decisions

### What I did
- Added `actions/cache@v4` for `node_modules` (root + all 3 workspace dirs) to both CI jobs
- Cache key includes Node version + `package-lock.json` hash — ensures correct modules per Node version
- `npm ci` only runs on cache miss (conditional `if: steps.cache-modules.outputs.cache-hit != 'true'`)
- Kept `setup-node`'s `cache: npm` as a fallback layer for cache misses (caches download tarballs)

### Key decisions

- Kept `cache: npm` on `setup-node` alongside `actions/cache` — belt-and-suspenders. When node_modules cache misses, the npm download cache still speeds up `npm ci`
