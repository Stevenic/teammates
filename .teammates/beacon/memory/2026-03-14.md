## Task: Episodic memory compaction system

Implemented the full episodic memory compaction system per Scribe's design doc (EPISODIC-COMPACTION.md). This gives teammates scalable episodic memory: daily logs compact into weekly summaries (kept 52 weeks), then into monthly summaries (kept permanently).

### Changes across both packages

**CLI package:**
- `types.ts` — Added `WeeklyLog` and `MonthlyLog` interfaces, `weeklyLogs` field to `TeammateConfig`
- `registry.ts` — Added `loadWeeklyLogs()` to discover weekly summaries from `memory/weekly/` (YYYY-Wnn.md pattern)
- `adapter.ts` — Injects 2 most recent weekly summaries into teammate prompts (between daily logs and roster)
- `compact.ts` (new) — Full compaction logic: `compactDailies()` groups by ISO week, creates weekly summaries with frontmatter, deletes source dailies; `compactWeeklies()` archives weeklies older than 52 weeks into monthly summaries
- `cli.ts` — Added `/compact [teammate]` command with spinner and per-teammate output; starts recall `watch` mode as background process on session start (if recall installed); kills watch process on exit; added `weeklyLogs` to all inline TeammateConfig constructions; added `compact` and `debug` to TEAMMATE_ARG_POSITIONS for wordwheel autocomplete
- All test files updated with `weeklyLogs: []`

**Recall package:**
- `indexer.ts` — Updated `collectFiles()` to skip raw daily logs from indexing (already in prompt context), added `memory/weekly/*.md` and `memory/monthly/*.md` collection
- `search.ts` — Implemented multi-pass retrieval: Pass 1 (recency) always includes N most recent weekly summaries; Pass 2 (semantic) does query-driven search; results merged with dedup and typed memory boost (1.2x); added `contentType` field and `classifyUri()` helper
- `cli.ts` — Added `watch` command using `fs.watch` with recursive option, debounced sync (2s), graceful shutdown on SIGTERM/SIGINT, JSON output mode

### Key decisions
- Raw daily logs are NOT indexed by recall — they're already loaded directly by adapter.ts (last 7)
- Recency pass reads weekly files directly from disk (not via Vectra index) for speed and reliability
- Typed memories get a 1.2x relevance boost over episodic summaries at the same semantic score
- Watch mode uses Node.js built-in `fs.watch` with recursive option — zero new dependencies
- Compaction produces structural summaries (concatenation with frontmatter) — agents can refine later
- Current week's dailies are never compacted (only completed weeks)
