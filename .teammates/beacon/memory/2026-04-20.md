## Task: Tabs polish ‚Äî drop [reply], coalesce refreshes
Two fixes from user feedback on tabs.

**Issue 1:** `[reply]` was redundant in tab model (reply is implicit ‚Äî just type in focused thread). Removed action from `displayAgentResult` and `displayCanceledInThread`; kept `[copy thread]`. Deleted dead `thread-reply-` click handler. Field `replyActionIdx` kept to avoid rename churn ‚Äî still tracks the thread-action line's feed index.

**Issue 2:** TUI slowed with concurrent tabs. Root cause: `refreshView()` called `app.refresh()` (synchronous full measure‚Üíarrange‚Üírender) with no coalescing ‚Äî ~100+ call sites √ó N concurrent agents = render storm. Fix: one-line change to `app.scheduleRefresh()` which uses `setImmediate` to coalesce per-tick. Safe because all mutations already `invalidate()`, and hot paths (`showPrompt`, banner animation) still use `app.refresh()` directly.
