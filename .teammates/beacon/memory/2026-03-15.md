## Task: Robust handoff detection

Fixed handoff detection in `parseMessageProtocol` — was only scanning first 5 lines for `TO:`, but agent output (Claude Code) buries the response deep in tool-call output. Now uses 3-pass detection:

1. **[handoff:] marker** — `[handoff:<teammate>]` anywhere in output (most explicit)
2. **Quick scan** — `TO:` in first 5 lines (clean output, existing behavior)
3. **Deep scan** — `TO: <known_teammate>` anywhere in output (catches buried responses)

Also updated the output protocol prompt to instruct agents to include the `[handoff:]` marker when handing off.

### Files changed
- `cli/src/adapters/cli-proxy.ts` — 3-pass parsing, new `findHandoffMarker` + `extractHandoffFromMarker` helpers
- `cli/src/adapter.ts` — Added [handoff:] instruction to output protocol

## Task: Code review — embedded handoff format switch

Reviewed the refactor switching from JSON protocol + single `handoff` field to fenced code block (`\`\`\`handoff`) format with multi-handoff support.

### What changed
- `types.ts` — `TaskResult.handoff?` → `handoffs: HandoffEnvelope[]`; removed `TaskAssignment.handoff`; removed `handoff_initiated`/`handoff_completed` events
- `orchestrator.ts` — Removed all handoff chain logic (cycle detection, max depth, auto-follow, pending-handoff state). Orchestrator now returns results directly; CLI layer handles handoff approval/queueing.
- `cli-proxy.ts` — New `findHandoffBlocks()` regex parser for \`\`\`handoff fences. Removed legacy JSON protocol, 3-pass detection, `findHandoffMarker`/`extractHandoffFromMarker`.
- `adapter.ts` — Output protocol instructs agents to use fenced handoff blocks
- `cli.ts` — `renderHandoffs()` UI with approve/reject/always-approve actions; strips handoff blocks from displayed output
- `orchestrator.test.ts` — Replaced 4 handoff chain tests with 1 test verifying handoffs returned in result
- `adapter.test.ts` — Updated protocol assertions
- `echo.ts` — Added `handoffs: []` to result

### Verdict

- Architecture improvement: handoff approval is now a UI concern (CLI), not orchestrator logic
- Multi-handoff support is a good addition — agents can hand off to multiple teammates at once
- Regex pattern `\`\`\`handoff\s*\n@(\w+)\s*\n([\s\S]*?)\`\`\`` is correct and handles the format well
