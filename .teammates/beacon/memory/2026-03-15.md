## Task: Robust handoff detection

Fixed handoff detection in `parseMessageProtocol` — was only scanning first 5 lines for `TO:`, but agent output (Claude Code) buries the response deep in tool-call output. Now uses 3-pass detection:

1. **[handoff:] marker** — `[handoff:<teammate>]` anywhere in output (most explicit)
2. **Quick scan** — `TO:` in first 5 lines (clean output, existing behavior)

## Task: Fix SQLite ExperimentalWarning leaking into terminal UI

User reported `ExperimentalWarning: SQLite is an experimental feature and might change at any time` with `[CLI subprocess]` prefix appearing in the terminal UI while talking to copilot.

### Root cause
The `@github/copilot-sdk`'s `CopilotClient` spawns a CLI subprocess. That subprocess uses Node.js's experimental `node:sqlite` API, which emits a warning to stderr. The SDK's `client.ts` (line 1244-1253) forwards ALL subprocess stderr to `process.stderr` prefixed with `[CLI subprocess]`, causing the warning to appear in the terminal.

### Fix
Added `NODE_OPTIONS=--disable-warning=ExperimentalWarning` to the environment of all subprocess spawns across both adapters:
1. **`copilot.ts`** — `ensureClient()` now builds an env object with the suppression flag and passes it to `CopilotClient` via the `env` option
2. **`cli-proxy.ts`** — `spawnAndProxy()` and `routeTask()` both add the flag to the spawn environment

The fix preserves any existing `NODE_OPTIONS` value (appends, doesn't replace).

### Key decisions
- Fix at the env level (not stderr interception) — clean, targeted, no side effects
- Applied to both adapters for consistency, even though the immediate bug is in copilot
- Only suppresses `ExperimentalWarning` — other warnings still pass through

- Changes are TypeScript-safe: just env object manipulation + passing existing `CopilotClient.env` option
