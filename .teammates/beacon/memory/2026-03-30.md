## Task: Fix Avalonia startup crash from invalid CornerRadius resource

Fixed the startup `System.InvalidCastException` introduced in the Avalonia shell resources after the tab-control refactor.

### Root cause
- `src\TeamMates\TeamMates\App.axaml` declared `DefaultCornerRadius` as an `x:Double`.
- Multiple `Border.CornerRadius` bindings in `MainView.axaml` consumed that resource through `DynamicResource`, so Avalonia tried to cast a `double` to `Avalonia.CornerRadius` during `TabControl` template application and crashed on startup.

### Key decisions
- Kept the shared token name `DefaultCornerRadius` and corrected its resource type to `<CornerRadius>` instead of changing all consuming bindings.
- Verified with a full `dotnet build src\TeamMates.slnx` so the desktop shell and shared project both compile cleanly after the fix.

### Files changed
- `src/TeamMates/TeamMates/App.axaml`
- `.teammates/beacon/memory/2026-03-30.md`

### Notes for next task
- When adding shared Avalonia design tokens, declare them with the concrete Avalonia type expected by the consuming property; `DynamicResource` does not coerce `double` into `CornerRadius`.
