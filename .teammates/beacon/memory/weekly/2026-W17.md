## Task: Fix `/clear` removing the tab bar
Bug: with only the Default tab present, `/clear` wiped the tab from the bar and required creating a new `/tab` before the original tab would redisplay.

### Root cause
`cmdClear` in `commands.ts` takes two paths:
- `threads.size > 1` → per-thread clear (clears focused feed only, tab preserved)
- else → full reset, which calls `threadManager.clear()` (wipes all threads, `nextThreadId` reset to 1)

The full-reset path never recreated the Default thread, so the tab bar went empty. The same seeding logic already runs at startup in `cli.ts` and in `cmdTab` when `threads.size === 0`.

### Fix
After `threadManager.clear()` and `orchestrator.reset()`, re-seed the Default thread with the same pattern used at startup:

```ts
const defaultThread = d.threadManager.createThread("Default");
d.threadManager.appendThreadEntry(defaultThread.id, {
  type: "user",
  content: "",
  timestamp: Date.now(),
});
```

`createThread` → `switchToThread` → `onTabsChanged?.()` fires, so the tab bar refreshes automatically.

### Files changed
- `packages/cli/src/commands.ts` — re-seed Default thread inside `cmdClear` full-reset path
