## Task: Episodic memory compaction system

Implemented the full episodic memory compaction system per Scribe's design doc (EPISODIC-COMPACTION.md). This gives teammates scalable episodic memory: daily logs compact into weekly summaries (kept 52 weeks), then into monthly summaries (kept permanently).

### Changes across both packages

**CLI package:**
- `types.ts` — Added `WeeklyLog` and `MonthlyLog` interfaces, `weeklyLogs` field to `TeammateConfig`
- `registry.ts` — Added `loadWeeklyLogs()` to discover weekly summaries from `memory/weekly/` (YYYY-Wnn.md pattern)
- `adapter.ts` — Injects 2 most recent weekly summaries into teammate prompts (between daily logs and roster)
- `compact.ts` (new) — Full compaction logic: `compactDailies()` groups by ISO week, creates weekly summaries with frontmatter, deletes source dailies; `compactWeeklies()` archives weeklies older than 52 weeks into monthly summaries
- `cli.ts` — Added `/compact [teammate]` command with spinner and per-teammate output; starts recall `watch` mode as background process on session start (if recall installed); kills watch process on exit; added `weeklyLogs` to all inline TeammateConfig constructions; added `compact` and `debug` to TEAMMATE_ARG_POSITIONS for wordwheel autocomplete
- All test files updated with `weeklyLogs: []`

**Recall package:**
- `indexer.ts` — Updated `collectFiles()` to skip raw daily logs from indexing (already in prompt context), added `memory/weekly/*.md` and `memory/monthly/*.md` collection
- `search.ts` — Implemented multi-pass retrieval: Pass 1 (recency) always includes N most recent weekly summaries; Pass 2 (semantic) does query-driven search; results merged with dedup and typed memory boost (1.2x); added `contentType` field and `classifyUri()` helper
- `cli.ts` — Added `watch` command using `fs.watch` with recursive option, debounced sync (2s), graceful shutdown on SIGTERM/SIGINT, JSON output mode

### Key decisions
- Raw daily logs are NOT indexed by recall — they're already loaded directly by adapter.ts (last 7)
- Recency pass reads weekly files directly from disk (not via Vectra index) for speed and reliability
- Typed memories get a 1.2x relevance boost over episodic summaries at the same semantic score
- Watch mode uses Node.js built-in `fs.watch` with recursive option — zero new dependencies
- Compaction produces structural summaries (concatenation with frontmatter) — agents can refine later
- Current week's dailies are never compacted (only completed weeks)

## Task: Fix SQLite ExperimentalWarning leaking into terminal UI

User reported `ExperimentalWarning: SQLite is an experimental feature and might change at any time` with `[CLI subprocess]` prefix appearing in the terminal UI while talking to copilot.

### Root cause
The `@github/copilot-sdk`'s `CopilotClient` spawns a CLI subprocess. That subprocess uses Node.js's experimental `node:sqlite` API, which emits a warning to stderr. The SDK's `client.ts` (line 1244-1253) forwards ALL subprocess stderr to `process.stderr` prefixed with `[CLI subprocess]`, causing the warning to appear in the terminal.

### Fix

1. **`copilot.ts`** — `ensureClient()` now builds an env object with the suppression flag and passes it to `CopilotClient` via the `env` option
2. **`cli-proxy.ts`** — `spawnAndProxy()` and `routeTask()` both add the flag to the spawn environment

The fix preserves any existing `NODE_OPTIONS` value (appends, doesn't replace).

### Key decisions
- Fix at the env level (not stderr interception) — clean, targeted, no side effects
- Applied to both adapters for consistency, even though the immediate bug is in copilot
- Only suppresses `ExperimentalWarning` — other warnings still pass through

## Task: Fix table column shrinking + ctrl+click URL on Windows

Two bugs reported by user.

### Bug 1: Table column shrinking too aggressive
Small columns were shrunk below their natural width by the proportional algorithm. E.g., "short" (5 chars) squeezed into a 4-char inner column, wrapping to "shor" + "t".

**Root cause:** First pass of proportional shrinking checked if a column's proportional share exceeded its natural width — but for small columns next to very large ones, the proportional share was tiny. Small columns weren't locked, then got clamped to MIN_COL (6) which was still too small.

**Fix:** Changed first pass to lock columns whose natural width is at or below the fair share (`budgetForCols / numCols`). Small columns keep their natural width; only oversized columns are proportionally shrunk.

### Bug 2: Ctrl+click URLs not opening on Windows
**Root cause:** Windows `start` command treats the first quoted argument as a **window title**. `start "https://example.com"` opens a blank cmd window titled with the URL instead of opening it in a browser.

**Fix:** Changed to `start "" "url"` (empty title, then quoted URL). Also added error callback to `execCb` to suppress unhandled error warnings.
