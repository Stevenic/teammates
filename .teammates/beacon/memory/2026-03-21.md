## Task: Fix selection highlight persisting after scroll

User reported that mouse-selected text highlights persist visually after scrolling, appearing stuck on screen.

### Root cause
Selection coordinates (`_selAnchor`, `_selEnd`) are stored as absolute screen positions. When the user scrolls, `_feedScrollOffset` changes and feed content shifts, but `_renderSelection()` still draws highlights at the original screen coordinates. The highlight appears frozen while content moves underneath it.

### What was done
- Added `clearSelection()` call inside `scrollFeed()` when not actively drag-selecting (`!this._selecting`)
- Added `clearSelection()` calls in the two scrollbar-initiated scroll paths (click-to-position and drag) which modify `_feedScrollOffset` directly without going through `scrollFeed()`
- This matches standard terminal behavior тАФ scrolling clears any active text selection

### Key decisions
- Clear-on-scroll (not adjust-on-scroll) тАФ simpler, matches terminal conventions, no risk of coordinate drift
- Guard with `!this._selecting` so auto-scroll during drag-to-select still works correctly
- Three sites covered: `scrollFeed()`, scrollbar click-to-position, scrollbar thumb drag

## Task: Recall query architecture тАФ Two-pass implementation

Built the full recall query architecture per Scribe's spec (`F-recall-query-architecture.md`). This addresses the chicken-and-egg problem: the pre-task query must retrieve relevant memories before the agent knows what's relevant.

### Phase 1 тАФ Pass 1 improvements (adapter-side, no LLM)

**1. Keyword extraction (`query-expansion.ts`)**
- `extractKeywords(text)` тАФ removes stopwords, filters short tokens, deduplicates
- `buildQueryVariations(taskPrompt, conversationContext?)` тАФ generates 1-3 queries:
  - Original task prompt (always)
  - Focused keyword query (if keywords differ meaningfully from original)
  - Conversation-derived topic query (extracts recent topic from history)

**2. Memory frontmatter scanning (`memory-index.ts`)**
- `scanMemoryCatalog(teammatesDir, teammate)` тАФ reads all typed memory files' frontmatter (name + description)
- `matchMemoryCatalog(teammatesDir, teammate, taskPrompt)` тАФ text-matches task prompt words against frontmatter fields, returns matching files' full content as SearchResults
- Scores in 0.85-0.95 range based on word overlap count
- The key insight from the spec: **the teammate's memory catalog is a cheap, no-LLM relevance signal**

**3. Multi-query fusion (`search.ts` тЖТ `multiSearch()`)**
- Fires primary query (full recency + semantic) + additional queries (semantic only, no duplicate weeklies)
- Merges catalog matches (from frontmatter text matching)
- Deduplicates by URI тАФ when same URI appears from multiple queries, highest score wins
- Returns top N results sorted by score

**4. Enhanced `queryRecallContext` (`adapter.ts`)**
- Now calls `buildQueryVariations()` to generate multi-query set
- Calls `matchMemoryCatalog()` for frontmatter text matching
- Calls `multiSearch()` with all query variations + catalog matches
- Falls back gracefully тАФ if conversation context isn't separate, uses taskPrompt for both

### Phase 2 тАФ Pass 2: Agent-driven recall (mid-task tool)

- Added `## Recall тАФ Memory Search Tool` section to every teammate prompt
- Documents the `teammates-recall search` CLI command with usage, flags, and tips
- Agents can now search iteratively mid-task: query тЖТ read result тЖТ refine query
- Supports cross-teammate search by omitting `--teammate` flag
- No MCP dependency тАФ works with existing CLI tool (MCP server deferred to S26)

### Open questions resolved
1. **Per-query result limit for Pass 2:** Trust the agent (no hard cap in prompt)
2. **Memory index:** Read on-demand per query (no startup cost, no stale cache)
3. **Cross-teammate search:** Supported via `--teammate` omission, documented in prompt

### Key decisions
- Multi-query fires the primary query with full recency pass, additional queries skip recency (avoids duplicate weekly summaries)
- Keyword extraction uses a hardcoded stopword list (~100 words) тАФ no external NLP dependency
- Memory catalog matching uses word overlap + substring matching (e.g., "goal" matches "project_goals")
- Catalog match scores (0.85-0.95) are slightly below semantic search (0.9+) so they don't dominate
- Conversation topic extraction splits on `**role:**` patterns and takes keywords from last 2 entries
- Pass 2 prompt section is always included (not conditional) тАФ every teammate should know about recall
