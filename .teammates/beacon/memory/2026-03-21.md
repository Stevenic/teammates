## Task: Fix selection highlight persisting after scroll

User reported that mouse-selected text highlights persist visually after scrolling, appearing stuck on screen.

### Root cause
Selection coordinates (`_selAnchor`, `_selEnd`) are stored as absolute screen positions. When the user scrolls, `_feedScrollOffset` changes and feed content shifts, but `_renderSelection()` still draws highlights at the original screen coordinates. The highlight appears frozen while content moves underneath it.

### What was done
- Added `clearSelection()` call inside `scrollFeed()` when not actively drag-selecting (`!this._selecting`)
- Added `clearSelection()` calls in the two scrollbar-initiated scroll paths (click-to-position and drag) which modify `_feedScrollOffset` directly without going through `scrollFeed()`
- This matches standard terminal behavior — scrolling clears any active text selection

### Key decisions
- Clear-on-scroll (not adjust-on-scroll) — simpler, matches terminal conventions, no risk of coordinate drift
- Guard with `!this._selecting` so auto-scroll during drag-to-select still works correctly
- Three sites covered: `scrollFeed()`, scrollbar click-to-position, scrollbar thumb drag
