## Task: Ctrl+click to open web links in browser

Added ctrl+click URL detection to the ChatView widget (consolonia) and wired it up in the CLI to open links in the default browser.

### What was done
1. **ChatView** (`packages/consolonia/src/widgets/chat-view.ts`):
   - Added `URL_REGEX` constant for `https?://` URL detection
   - Added `_screenToFeedRow` map (tracks which row within a multi-row wrapped line each screen Y corresponds to)
   - Cached `_feedX` and `_contentWidth` from render pass for position math
   - Added `_extractFeedLineText()` helper to get plain text from a feed line (handles both plain strings and StyledSpan segments)
   - Added `_findUrlAtOffset()` helper for precise URL hit-testing at a character position
   - Added ctrl+left-click handler: detects URLs in the clicked feed line, resolves which URL based on click position (accounts for line wrapping), emits `"link"` event with the URL
2. **CLI** (`packages/cli/src/cli.ts`):
   - Added `"link"` event listener that opens URLs via platform-appropriate command (`open` on macOS, `start` on Windows, `xdg-open` on Linux)

### Key decisions
- Ctrl+click is checked before regular action click — so ctrl+clicking an action line with a URL opens the URL, not the action
- For single-URL lines, opens immediately without position math
- For multi-URL lines, calculates approximate character offset from click position (row * contentWidth + col) and finds the URL at that offset
- Falls back to first URL on the line if position math doesn't hit exactly

## Task: Fix table column shrinking + ctrl+click URL on Windows

Two bugs reported by user.

### Bug 1: Table column shrinking too aggressive
Small columns were shrunk below their natural width by the proportional algorithm. E.g., "short" (5 chars) squeezed into a 4-char inner column, wrapping to "shor" + "t".

**Root cause:** First pass of proportional shrinking checked if a column's proportional share exceeded its natural width — but for small columns next to very large ones, the proportional share was tiny. Small columns weren't locked, then got clamped to MIN_COL (6) which was still too small.

**Fix:** Changed first pass to lock columns whose natural width is at or below the fair share (`budgetForCols / numCols`). Small columns keep their natural width; only oversized columns are proportionally shrunk.

### Bug 2: Ctrl+click URLs not opening on Windows
**Root cause:** Windows `start` command treats the first quoted argument as a **window title**. `start "https://example.com"` opens a blank cmd window titled with the URL instead of opening it in a browser.

**Fix:** Changed to `start "" "url"` (empty title, then quoted URL). Also added error callback to `execCb` to suppress unhandled error warnings.
