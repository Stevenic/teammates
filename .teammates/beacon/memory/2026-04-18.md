## Task: Implement Thread UX v2 (Tab-Style Task Switching)
Implemented the full Thread UX v2 spec from `.teammates/scribe/docs/specs/F-thread-ux-v2.md`. Major architectural refactor replacing the single-shared-feed threading model with per-thread tab-based feeds. Bumped all packages from 0.8.1 to 0.9.0.

### Architecture changes
- **Per-thread FeedStore**: Each thread now owns its own `FeedStore` instance. Cross-thread index shifting is eliminated ‚Äî the #1 source of complexity in the old model.
- **FeedAdapter**: New `packages/cli/src/feed-adapter.ts` wraps a FeedStore and provides the `ThreadFeedView` interface. Replaces ChatView as the mutation target for thread content.
- **ThreadBar widget**: New `packages/consolonia/src/widgets/thread-bar.ts` ‚Äî 1-row tab bar with thread names, unread ‚óè badges, working ‚óé indicators, [x] close buttons, and [+] new thread button.
- **ChatView docked bar + store swapping**: Added `dockedBar` option for pinned widgets between banner and feed. Added `setStore()` / `store` getter for swapping active FeedStore when switching tabs.

### Behavior changes per spec
- Session starts with thread #1, tab bar hidden (single-thread mode)
- `/thread [description]` creates a new tab and switches to it
- `#id` alone switches focus (no message required)
- `@mention` dispatches within the current thread (no longer creates new threads)
- `/clear` clears only the focused thread's feed
- `/close [#id]` removes a thread tab (thread #1 permanent)
- `/threads` lists all threads with status
- Tab bar appears when >1 thread exists, docks at top

### Files created
- `packages/consolonia/src/widgets/thread-bar.ts` -- ThreadBar widget
- `packages/cli/src/feed-adapter.ts` -- FeedStore-backed ThreadFeedView adapter

### Files changed (major)
- `packages/consolonia/src/widgets/chat-view.ts` -- dockedBar, setStore(), store getter
- `packages/consolonia/src/index.ts` -- export ThreadBar, ThreadBarTab
- `packages/cli/src/thread-manager.ts` -- complete rewrite for per-thread feeds
- `packages/cli/src/cli.ts` -- ThreadBar wiring, routing changes, tab switching
- `packages/cli/src/commands.ts` -- /thread, /close, /threads, per-thread /clear
- `packages/cli/src/activity-manager.ts` -- uses FeedAdapter instead of ChatView
- `packages/cli/src/feed-renderer.ts` -- uses adapter for threaded results
- All package.json files + settings.json -- version 0.9.0

### Decisions
- Kept ThreadContainer for per-thread index management (still useful for insert-point tracking within a thread's own feed)
- FeedAdapter wraps FeedStore and provides the same interface as ChatView's feed methods ‚Äî lets ThreadContainer work with any feed, active or background
- Activity tracking uses per-thread adapters so activity lines go into the right thread even when viewing a different tab

## Task: Fix tab bar not appearing on /tab, add Default tab and [x] on all tabs
User reported `/tab` didn't show the tab bar. Root cause: when no threads exist, `/tab` creates thread #1 as the new tab (since `nextThreadId` starts at 1), leaving only 1 thread ‚Äî tab bar stays hidden (`tabs.length > 1`).

### Fixes
- **cmdTab** (commands.ts) ‚Äî if no threads exist, create thread #1 as "Default" first, then create the new tab as thread #2
- **[+] handler** (cli.ts) ‚Äî same fix for the ThreadBar's [+] button
- **thread-bar.ts** ‚Äî removed `tab.id > 1` gate on [x] close button; all tabs now show [x]
- **closeThread** (thread-manager.ts) ‚Äî changed from "can't close #1" to "can't close last remaining tab"
- **cmdClose** (commands.ts) ‚Äî updated error message and removed hardcoded `targetId === 1` check
