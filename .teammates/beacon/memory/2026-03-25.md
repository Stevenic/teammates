## Task: Non-blocking system tasks + debug enhancements (4 changes)

User requested 4 changes to the CLI.

### 1. Non-blocking system task lane
System-initiated tasks (compaction, summarization, wisdom distillation) now run concurrently without blocking user tasks. An agent can run 0+ system tasks and 0-1 user tasks simultaneously.

**What was done:**
- Added `isSystemTask()` helper — compact, summarize, and `system: true` agent tasks are system tasks
- Added `systemActive` map to track running system tasks
- `kickDrain()` now extracts system tasks from the queue and fires them via `runSystemTask()` before processing user tasks
- `runSystemTask()` runs each system task independently, tracks in `activeTasks` for progress bar display
- Wisdom distillation queued by `runCompact()` now uses `system: true` flag and calls `kickDrain()` immediately

### 2. System tasks suppress feed output (errors only)
- `runSystemTask()` uses `silentAgents` to suppress orchestrator events for agent-based system tasks
- Compact tasks run with `silent=true` (existing behavior)
- Errors always show in feed with `(system)` label
- `/status` shows active system tasks with `⟳ (system)` prefix

### 3. `/debug <teammate> <optional debug focus>`
- `cmdDebug()` now parses args as `<teammate> <focus...>` where focus is everything after the teammate name
- Focus text is passed to `queueDebugAnalysis()` and inserted into the analysis prompt as "Focus your analysis on: ..."
- Focus is also shown in the feed when queuing the analysis
- Usage updated to `/debug [teammate] [focus]`

### 4. Full prompt in debug logs
- Added `fullPrompt?: string` to `TaskResult` in types.ts
- cli-proxy and copilot adapters now set `result.fullPrompt` after building the prompt
- `writeDebugEntry()` includes a "## Full Prompt" section in the debug log when available
- `lastTaskPrompts` now stores the full prompt (instead of just the task text) for `/debug` analysis

### Key decisions
- System tasks use a unique `taskId` (`sys-<teammate>-<timestamp>`) to allow multiple concurrent system tasks per agent
- `drainAgentQueue` now filters out system tasks with `!this.isSystemTask(e)` as a safety net
- QueueEntry got `system?: boolean` on the agent variant (compact and summarize are inherently system tasks)

## Task: Fix system tasks blocking user task progress bar

User reported actual task progress not showing after the system task filtering changes.

### Root cause
`silentAgents` was agent-level, not task-level. When `runSystemTask()` added a teammate to `silentAgents` for wisdom distillation, it suppressed ALL events for that agent — including user task `task_assigned` events. During startup, all teammates got wisdom distillation queued simultaneously, putting them all in `silentAgents`. Any user task assigned during that window (~30s-2min per agent) had its progress bar completely suppressed.

### What was done
- Added `system?: boolean` to `TaskAssignment` and `TaskResult` in types.ts
- `runSystemTask()` now passes `system: true` to `orchestrator.assign()` for both summarize and system agent tasks — removed `silentAgents` usage entirely
- `orchestrator.ts` propagates `system` flag from assignment to result
- `handleEvent` checks `event.assignment.system` / `event.result.system` — skips `activeTasks` management for system tasks only, user task events flow normally even for the same agent
- Removed `startupMaintenance` setProgress calls (system-level messages, shouldn't be in progress bar)
- Guarded `runCompact` error handler's `setProgress(null)` with `!silent`
- `silentAgents` retained only for the defensive retry in `drainAgentQueue`

### Key decisions
- Task-level filtering (via `system` flag on events) replaces agent-level filtering (`silentAgents`) for system tasks
- This allows system and user tasks for the same agent to run concurrently without interference
- `silentAgents` still used for defensive retry (that's a different pattern — temporary, per-drain-loop)

### Build & test
- TypeScript compiles cleanly (strict mode)
- 224 CLI tests pass

### Files changed
- `packages/cli/src/types.ts` — `system?: boolean` on TaskAssignment and TaskResult
- `packages/cli/src/orchestrator.ts` — propagate system flag to result
- `packages/cli/src/cli.ts` — `handleEvent` system task filtering, `runSystemTask` uses system flag instead of silentAgents, removed startupMaintenance progress calls, guarded runCompact error setProgress
