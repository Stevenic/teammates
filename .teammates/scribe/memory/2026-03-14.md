# 2026-03-14

## Implemented three-tier memory system (WISDOM)

Major framework overhaul replacing the two-layer memory system (MEMORIES.md + daily logs) with a three-tier system:

- **Tier 1 — Daily Logs** (`memory/YYYY-MM-DD.md`) — unchanged, raw append-only
- **Tier 2 — Typed Memories** (`memory/<type>_<topic>.md`) — individual files with frontmatter, four types: user, feedback, project, reference
- **Tier 3 — WISDOM.md** — distilled principles from compacted memories, read second after SOUL.md

Key decisions:
- Compaction: manual via `/compact` + automatic every 7 days
- WISDOM.md is a single file (not individual files)
- Compacted memories are deleted (not archived)
- Read order: SOUL.md → WISDOM.md → daily logs → memories on-demand
- Wisdom is personal per teammate (no group wisdom)
- Optional INDEX.md for recall support, but not required
