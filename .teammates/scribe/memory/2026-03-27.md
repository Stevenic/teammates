## Designed interrupt-and-resume mechanism for agent timeouts
- User proposed: since we log all tool calls per session, on timeout/hang we could kill the agent, pick up the conversation log, append the user's interruption text, and pass it as a new prompt with original context
- This is essentially checkpoint/restore using the conversation log as state
- Analyzed the design: agent-agnostic (works with any agent that has tool call logs), preserves all progress, adds human steering mid-task
- Key design considerations: log format must be replayable as conversation, token budget on resume (may need log compaction), idempotency (resumed agent needs to know what's already done)
- Proposed two-part feature: manual `/interrupt <agent> <message>` command + automatic timeout-triggered interrupt at 80% of timeout
- Open questions: where it lives (adapter vs protocol), log compaction strategy, multi-interrupt support
- Awaiting user decision on whether to formalize as a spec

## Wrote interrupt-and-resume spec + handoff to Beacon
- User approved — explored full codebase (adapter, orchestrator, cli.ts, types.ts, debug logging) to understand the implementation surface
- Wrote comprehensive spec at `.teammates/scribe/docs/specs/F-interrupt-and-resume.md`
- **3 phases:** Phase 1 = manual `/interrupt <agent> [message]`, Phase 2 = auto-interrupt at 80% timeout (kill + resume instead of hard kill), Phase 3 = log compaction for long sessions (>8k tokens)
- **Key adapter change:** expose `killAgent()` method + track `activeProcesses` map on CliProxyAdapter, so REPL can kill a running agent on demand
- **Log parsing per preset:** Claude `--debug-file` (structured JSON with tool calls), Codex `--json` JSONL, others raw stdout capture
- **Resume prompt structure:** original full prompt + condensed conversation log (tool calls/results without file contents) + user interruption text + resume instructions ("don't repeat done work, check filesystem")
- **Token budget:** ~21-25k total for resume prompt — fits comfortably
- **Edge cases covered:** idempotency (check filesystem before re-writing), partial file writes, multiple interrupts, handoff mid-task, non-Claude agents
- Updated CROSS-TEAM.md with spec pointer
- Handed off to @beacon for implementation (adapter changes, log parser, `/interrupt` command, auto-interrupt timer)
