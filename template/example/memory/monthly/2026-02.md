Major theme: API v2 migration and database performance.

- Shipped API v2 with versioned endpoints (`/v2/projects`, `/v2/users`). v1 endpoints remain active with deprecation headers. Migration guide published in the project wiki.
- Migrated from raw SQL queries to Prisma ORM across all endpoints. Reduced query-related bugs by eliminating hand-written joins.
- Implemented rate limiting middleware (100 req/min per API key, configurable per tier). Added rate limit headers to all responses.
- Set up staging database with production-like data (anonymized) for integration testing. This replaced the previous mock-based approach after the Q1 incident.
